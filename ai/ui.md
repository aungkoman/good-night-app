# Product Vision — Good Night

We are developing a Flutter application called **"Good Night."**

The purpose of the app is simple:

> **Help users immediately start listening to a Buddhist Dhamma talk (တရားတော်) as soon as they open the app, with zero friction.**

This is **not** a music app, podcast app, or media library.

The primary experience is **instant listening**.

---

# Core User Experience

When the user opens the app:

* The app should immediately begin playing a recommended တရားတော်.
* No setup.
* No onboarding.
* No category selection.
* No speaker selection.
* No asking the user what they want.

The user should hear a Dhamma talk within a few seconds of opening the app.

Think of it as:

> Open → Listen → Relax

The experience should feel calm, peaceful, and effortless.

---

# First Launch Experience

On the very first launch:

* Do **not** ask the user to choose favorite monks or categories.
* Do **not** display a full-screen library.
* Do **not** interrupt the experience.

Instead:

1. Show a beautiful home screen.
2. Start playing today's recommended တရားတော် automatically.
3. Display elegant playback controls.
4. Allow the user to continue listening immediately.

---

# Primary Screen

The Home screen should focus on the currently playing တရားတော်.

Large artwork or calming illustration.

Display:

* Current title
* Speaker
* Playback progress
* Play / Pause
* Skip Forward
* Skip Backward
* Playback speed
* Sleep Timer
* Favorite button

This screen should feel peaceful rather than busy.

---

# Secondary Features

These are important but secondary to listening.

Include:

* Search
* Browse all တရားတော်
* Categories
* Speakers
* Recently Played
* Favorites
* Downloads (optional)
* Continue Listening

These should never distract from the primary listening experience.

---

# Navigation

Keep navigation simple.

Suggested tabs:

* Home (Now Playing)
* Library
* Search
* Favorites
* Settings

The Home tab should always open first.

---

# Search Experience

Provide fast, intuitive search.

Users should be able to search by:

* Speaker
* Title
* Topic
* Keywords

Search should be instant and responsive.

---

# UI & UX Goals

The app should evoke:

* Peace
* Calm
* Simplicity
* Comfort
* Mindfulness
* Focus

Avoid visual clutter.

Avoid excessive colors.

Avoid overwhelming menus.

The interface should disappear into the background, allowing the user to focus on listening.

---

# Design Inspiration

Imagine if Apple designed a meditation app specifically for listening to Buddhist Dhamma talks.

The experience should be:

* Beautiful
* Minimal
* Premium
* Smooth
* Elegant
* Emotionally calming

Every animation should feel gentle.

Every interaction should reduce cognitive load.

---

# Success Criteria

The app is successful if a first-time user can:

1. Open the app.
2. Hear a တရားတော် within 3 seconds.
3. Instantly understand playback controls.
4. Continue listening without making any decisions.
5. Discover additional content later through Search or Library if desired.

The core philosophy is:

> **"Open the app. Hear the Dhamma. Everything else is secondary."**
